# SwarmForge：去中心化 AI 训练网络

## 一句话

先建网络，大家一起训练模型；模型够强后，开放全球推理服务。参考以太坊：节点贡献算力挖矿（训练），获得 Token 激励，模型是网络的共同资产。

## 两个阶段，一个逻辑

```
以太坊:
  先建矿工网络（PoW 挖矿）→ 网络有了共识 → 开放智能合约/DeFi → 生态繁荣

SwarmForge:
  先建训练网络（节点贡献算力训练模型）→ 模型够强 → 开放全球推理服务 → 生态繁荣

不是先做产品再找用户。
是先建网络，网络本身就是产品。
```

### Stage 1：训练网络（挖矿阶段）

```
全球节点加入网络 → 贡献算力做训练任务 → 获得 $FORGE Token
    │
    ▼
训练任务:
  - 蒸馏采集: 调用大模型 API 生成高质量数据
  - RL 采集: 本地 7B 做题 → sandbox 验证 → 生成训练轨迹
  - LoRA 微调: Tier 1/2 节点参与分布式训练
    │
    ▼
训练数据 + 算力 → 模型不断升级 → 网络资产增值
```

### Stage 2：推理网络（服务阶段）

```
模型能力达标后，开放全球推理服务:
    │
    ▼
用户发送代码请求 → 网络中多个节点协作推理 → 返回结果
  - 东京节点生成代码
  - 柏林节点写测试
  - 圣保罗节点修复
  - 2 秒拿到结果
    │
    ▼
推理请求消耗 $FORGE Token（Gas 费）
  → Token 流向提供推理的节点
  → 节点有持续收入动力
```

## 为什么这个顺序

| | 先做产品再建网络（❌ 旧思路） | 先建网络再做产品（✅ 新思路） |
|---|---|---|
| **冷启动** | 靠一个人训练模型，能力有限 | 万人一起训练，数据量级碾压 |
| **模型质量** | 受个人算力/预算限制 | 网络算力无上限 |
| **社区** | 用户是消费者，被动 | 节点是建设者，主动 |
| **经济** | 免费发放，没有闭环 | Token 激励，自循环 |
| **对标** | 像做一个 App | **像建一条链** |

| ETH | SwarmForge |
|-----|-----------|
| 矿工/验证者 | 训练节点 |
| 挖矿（PoW/PoS）| 训练任务（蒸馏/RL/微调）|
| ETH Token | $FORGE Token |
| Gas 费（交易） | Gas 费（推理请求）|
| 智能合约 | 专家模型（LoRA）|
| EVM | 7B 基座 + 管线协议 |
| 链上状态 | 模型权重（共同资产）|
| 节点质押 | 节点质押 $FORGE → 获得训练/推理资格 |
| 出块奖励 | 训练奖励（贡献数据/算力 → 获得 $FORGE）|
| 链的价值 = 生态 | 网络的价值 = 模型能力 |

### Token 分配

| 用途 | 占比 | 说明 |
|------|------|------|
| 训练挖矿奖励 | 60% | 贡献训练数据/算力的节点获得 |
| 推理服务奖励 | 20% | 提供推理服务的节点获得 |
| 核心开发 | 10% | 协议开发、基础设施 |
| 生态基金 | 10% | 社区治理、合作伙伴 |

### 为什么 Token 有价值

```
$FORGE 的价值 = 网络中模型的能力

模型越强 → 推理服务越有价值 → 推理需要 Gas → $FORGE 需求 ↑ → 价值 ↑

类比:
  ETH 的价值 = 以太坊网络的生态价值
  $FORGE 的价值 = SwarmForge 模型的能力价值

网络训练出的模型是所有节点的共同资产。
模型越强，Token 越值钱，越多人加入训练。
这是正向飞轮。
```

## 飞轮效应


## 终极目标

**SwarmForge 不是一个模型，是一条链。**

- 节点是矿工，贡献算力训练模型
- 模型是链上资产，所有节点共同拥有
- Token 是燃料，驱动训练和推理
- 网络越大，模型越强，Token 越有价值

先建网络，再做服务。
先有矿工，再有生态。
先训练模型，再服务全球。

模型会过时，网络不会。
