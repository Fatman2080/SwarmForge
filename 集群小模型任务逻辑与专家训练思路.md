# SwarmForge 网络：任务逻辑与专家训练思路

## 核心理念

**先建网络，大家一起训练模型。模型够强后，开放全球推理服务。**

不是先做产品找用户。是先建训练网络，让全球节点一起贡献算力训练模型。当模型能力足够强时，再开放推理服务，让模型为所有人服务。经济模型参考以太坊：节点=矿工，训练=挖矿，$FORGE=激励。

## 4. 三维专家体系

**维度 1 — 能力（7 种）**

Web-API / 数据处理 / 系统编程 / ML-AI / 前端

**维度 3 — 语言（4 种）**

Python / TypeScript / Rust / Go

### 6 阶段管线 — 全球节点协作

```
用户: "写一个 REST API 用户认证" → 消耗 5 $FORGE Gas

Stage 1: 并行生成 (~1s)
  🇯🇵 节点A [gen-web] → 候选 1,2,3   ← 获得 0.5 $FORGE
  🇩🇪 节点B [gen-web] → 候选 4,5,6   ← 获得 0.5 $FORGE
  🇧🇷 节点C [gen-web] → 候选 7,8,9   ← 获得 0.5 $FORGE

Stage 2: 测试生成 (与 Stage 1 并行)
  🇰🇷 节点D [test-web] → 测试用例     ← 获得 0.5 $FORGE

Stage 3: Sandbox 验证 (~0.5s)
  9 候选 × (原始测试 + 辅助测试) → 通过: 6 个

Stage 4: 修复 (~0.5s, 仅需时触发)
  🇺🇸 节点E [fix-web] → 修复失败候选   ← 获得 0.7 $FORGE

Stage 5: 审查选择 (~0.3s)
  🇮🇳 节点F [review-web] → 选最优      ← 获得 0.5 $FORGE

Stage 6: 最终验证
  用户本地 sandbox → ✅

总延迟: ~2 秒。参与节点: 6 个。Gas: 5 $FORGE。
```

### 为什么网络管线优于 Opus 单次

| 维度 | Opus 4.6 单次 | SwarmForge 网络管线 |
|------|-------------|-------------------|
| 多样性 | 1 个回答 | 9+ 候选 |
| 验证 | 无 | sandbox 验证 |
| 修复 | 无 | 自动修复 |
| 费用 | $0.01-0.10 | 极低 $FORGE Gas |
| 单点故障 | Anthropic 宕机 | 自动替换节点 |
| 所有权 | Anthropic | 所有节点 |
| 进化 | 公司决定 | 网络驱动，持续训练 |

### 管线参数

| 任务复杂度 | 候选数 | 节点 | 延迟 | Gas |
|-----------|--------|------|------|-----|

## 6. 数据质量与共识

### 训练数据验证 = SwarmForge 的共识机制

```
以太坊: 矿工出块 → 其他节点验证 → 达成共识 → 写入链
SwarmForge: 矿工出数据 → 其他节点验证 → 达成共识 → 写入训练集

验证流程:
1. 矿工 A 提交 RL 数据: {code, sandbox_result: pass}
2. 随机选 2 个验证矿工重跑 sandbox
3. 2/3 一致 → 数据入库 → 矿工 A 获得 $FORGE
4. 不一致 → 拒绝 → 矿工 A 扣信誉

SFT 数据额外验证:
5. Tier 1 节点用 32B 抽检 10%
6. 思考过程质量打分 → 低质量拒绝
```

### 防攻击

| 攻击 | 防御 | 惩罚 |
|------|------|------|
| 批量提交垃圾 RL | sandbox 验证 + 冗余 | 扣质押 10% |
| 伪造 sandbox 结果 | 验证矿工重跑 | 扣质押 30% |
| SFT 数据低质量 | Tier 1 抽检 | 降信誉 + 降任务优先级 |
| 女巫（多假节点） | 质押门槛 + 设备指纹 | 全额没收 |
| 投毒训练数据 | 版本隔离 + 训练前审计 | 永久封禁 |

## 10. ETH 经济模型详解

### Token 价值逻辑

```
为什么 $FORGE 有价值:

1. 训练价值（Stage 1）
   矿工贡献算力训练出强模型 → 模型是网络共同资产
   $FORGE = 对网络贡献的凭证
   类比: 早期 ETH 矿工持有的 ETH

2. 使用价值（Stage 2）
   用户使用推理服务需要消耗 $FORGE Gas
   模型越强 → 使用需求越大 → $FORGE 需求 ↑
   类比: ETH Gas fee

3. 治理价值
   $FORGE 持有者投票决定:
     - 训练什么专家
     - 何时开放推理网络
     - 奖励参数调整
   类比: ETH DAO 治理
```

### 经济闭环

```
Stage 1 闭环（训练期）:
  矿工质押 $FORGE → 参与训练 → 获得更多 $FORGE
  $FORGE 在矿工间流通和交易
  Token 价值 = 对未来推理服务的预期

Stage 2 闭环（服务期）:
  用户买 $FORGE → 消耗 Gas 推理 → Gas 分给矿工/节点
  矿工同时继续训练 → 模型升级 → 服务更强 → 更多用户
  Token 价值 = 实际推理服务价值

完整闭环:
  训练 ←→ Token ←→ 推理 ←→ 用户 ←→ 需求 ←→ 训练
```
